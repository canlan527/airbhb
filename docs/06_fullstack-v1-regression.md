# Airbhb Fullstack V1 回归验收记录

## 检查时间

2026-04-23

## 当前阶段

Fullstack V1 已完成 Phase 0 验收，项目进入 Phase 6 样式体系收敛阶段。

## 验收范围

本轮验收以 `docs/fullstack-v1.md` 为最新业务和技术基线，覆盖从原前端仿站升级到全栈版本后的核心链路：

- React/Vite 前端应用
- NestJS API 应用
- Prisma + PostgreSQL 数据模型
- 用户注册登录
- 首页、全部房源、房源详情旧页面兼容
- 个人中心
- 用户发布房源
- 管理后台
- 收藏、浏览历史、订单和模拟支付
- 平台自营房源创建与房源审核管理

## 验收结论

Phase 0 验收完成。当前版本已经具备可继续迭代的 Fullstack V1 基线：

- 前端和后端已经整理为 `apps/web` + `apps/api` 的 pnpm workspace 结构
- Vite 前端构建链路已经替代 CRA/CRACO
- NestJS 后端模块已经覆盖 Auth、Houses、Orders、Favorites、Histories、Admin
- Prisma schema 已经包含用户、房源、订单、收藏和浏览历史核心模型
- 用户端和管理端已经形成一条可演示的全栈业务闭环
- 原首页和房源列表所需的兼容接口已经由后端提供
- 新增业务页面已经开始使用 SCSS/Tailwind 方向的样式体系

## 已确认页面

### 前台页面

- `/home`：首页仍作为主入口，兼容原首页运营模块
- `/entire`：全部房源列表可继续承接原分页和房源卡片展示
- `/detail/:id`：详情页保留图片展示、预览、预订和支付交互
- `/profile`：个人中心展示订单、收藏、历史记录和我的房源
- `/publish-house`：普通用户可以提交房源进入审核流程

### 后台页面

- `/admin`：管理员可以查看平台看板、房源、订单和用户
- 管理员可以新增平台自营房源
- 管理员可以调整房源状态
- 管理员可以调整订单和用户状态
- 管理员可以查看房源审核详情

## 已确认业务链路

### 用户链路

1. 用户注册或登录
2. 用户浏览首页、列表页和详情页
3. 用户收藏房源
4. 用户产生浏览历史
5. 用户创建订单并模拟支付
6. 用户进入个人中心查看订单、收藏、历史和房源
7. 用户发布房源后成为房东角色

### 管理员链路

1. 管理员登录后进入 `/admin`
2. 查看统计看板
3. 查看用户发布房源
4. 审核、上架、驳回或下架房源
5. 新增平台自营房源
6. 管理订单状态
7. 管理用户状态

### 数据链路

- 用户密码由后端加密保存
- 登录后通过 JWT 访问受保护接口
- 普通用户接口通过鉴权保护
- 管理后台接口通过管理员角色保护
- 房源状态影响前台公开展示
- 已支付订单金额可进入后台成交金额统计
- 有历史订单的房源下架不应破坏历史订单数据

## 构建与运行基线

常用本地命令：

```bash
pnpm db:up
pnpm prisma:migrate
pnpm prisma:seed
pnpm dev:api
pnpm dev:web
```

构建验证命令：

```bash
pnpm build:api
pnpm build:web
```

前端默认地址：

```text
http://localhost:5173
```

后端默认地址：

```text
http://localhost:3001/api
```

## Phase 6 启动标准

由于 Phase 0 已完成，当前可以开始 Phase 6 的第一轮样式迁移：

- 新业务页面统一页面容器、头部、面板、卡片、表格和响应式规则
- 保留 Ant Design 组件，不大规模替换交互控件
- 保留旧页面 styled-components，先不影响首页、列表和详情页
- 新增或调整样式时优先沉淀到共享样式层，避免每个页面重复维护
